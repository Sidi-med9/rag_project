Category: Security and Accounts
Title: Implementing Secure SSH Key Authentication on Remote Servers
ID: SEC_06
Symptoms: Brute-force attacks targeting standard root passwords on public-facing cloud servers.
Resolution:
1. Generate an asymmetric key pair on the local workstation terminal: ssh-keygen -t ed25519
2. Export the public key securely over to the remote destination server: ssh-copy-id user@server-ip
3. Log into the remote terminal, open configuration file via editor: sudo nano /etc/ssh/sshd_config
4. Find the configuration line 'PasswordAuthentication' and set its value to 'no'.
5. Locate 'PermitRootLogin' and adjust to 'prohibit-password' or 'no'.
6. Save the settings file and safely restart the secure shell daemon: sudo systemctl restart ssh
