Category: Security and Accounts
Title: Enforcing Corporate Password Complexity and Lifecycle Rules
ID: SEC_04
Symptoms: Systems require upgrades to combat weak user credentials and brute force guessing methods.
Resolution:
1. Access the Group Policy Management Editor for the domain network.
2. Browse to Computer Configuration -> Windows Settings -> Security Settings -> Account Policies -> Password Policy.
3. Change 'Minimum password length' to a baseline requirement of 14 characters.
4. Enable the rule 'Password must meet complexity requirements', forcing combinations of uppercase, lowercase, numbers, and symbols.
5. Set 'Maximum password age' to expire keys every 90 days.
6. Configure account lockout thresholds to trip after 5 consecutive bad login attempts.
