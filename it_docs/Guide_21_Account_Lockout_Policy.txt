Category: Security and Accounts
Title: Unlocking Domain Accounts and Resolving Recurrent Lockouts
ID: SEC_01
Symptoms: Corporate users cannot log in, receiving notifications that their account has been locked due to repeated bad password attempts.
Resolution:
1. Access the Active Directory Users and Computers management tool on the domain controller.
2. Find the locked user profile, open Properties, and click the Account tab.
3. Check the box labeled 'Unlock account' and commit the changes.
4. If lockouts happen repeatedly within minutes, open the Account Lockout Status (LockoutStatus.exe) diagnostic tool.
5. Trace the source machine sending bad credentials; check for cached stale passwords in Windows Credential Manager or automated background scripts.
