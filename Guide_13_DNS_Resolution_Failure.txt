Category: Networking and Communications
Title: Troubleshooting Local DNS Server Resolution Timeouts
ID: NET_03
Symptoms: Web browsers report 'Server DNS Address Could Not Be Found' errors while internet access remains active via direct IPs.
Resolution:
1. Clear the local system DNS resolver cache on Windows: ipconfig /flushdns
2. On Linux, clear the systemd-resolved cache: sudo systemd-resolve --flush-caches
3. Open local adapter properties and shift DNS assignment away from ISP defaults.
4. Input public DNS alternatives: 8.8.8.8 for Google DNS or 1.1.1.1 for Cloudflare DNS.
5. Verify domain name lookups using the network diagnostic terminal command: nslookup google.com
