Category: Networking and Communications
Title: Preventing Remote Workstation VPN Disconnections
ID: NET_04
Symptoms: Virtual Private Network (VPN) tunnels close abruptly, dropping access to private network resources.
Resolution:
1. Verify internet connection stability over a physical line to rule out standard Wi-Fi drops.
2. Open the corporate VPN client software configuration and switch the tunnel protocol from UDP to TCP.
3. Disable local firewall rules temporarily to check if protocol traffic like IPsec or OpenVPN is blocked.
4. Modify the network adapter Maximum Transmission Unit (MTU) size down to 1350 to avoid packet fragmentation over home routers.
5. Check if double NAT conditions are causing tunnel state degradation.
